# 4視点デバッグフィードバック

Last updated: 2026-06-12

## 目的

スマホアプリとしてのリリース前に、同じ画面を「プレイヤー」「教育者」「配信者」「視聴者」の4視点で確認し、人気につながる改善とリリース前リスクを分けて管理します。

## 検査環境

- 390x844相当のスマホ縦画面。
- 静的HTML/CSS/Canvas版。
- 確認対象: タイトル、トップメニュー、デッキ、パック、設定、60秒プレイ、リザルト、共有画像、オフライン読み込み。
- 参照スクショ: `screenshot-menu-score-goal.png`, `screenshot-mobile.png`, `screenshot-result.png`, `result-share-preview.png`

## Findings

### P1 プレイヤー目線: 初回ホームの情報量を下げ、1プレイ開始を主導線にする
- 見える症状: スタミナ、日課、スコア目標、ストリーク、石が初回から並ぶと「まず何をすればいいか」が弱くなる。
- 再現手順: ローカル保存を消してタイトルからトップメニューを開く。
- なぜ重要か: 新規ユーザーは初回の数秒で継続判断をするため、遊ぶ前のタスク感を減らす必要がある。
- 対応状況: 対応済み。`tutorialComplete=false` の間は日課/スコア目標/今日のことば/今日の差し入れを隠し、デッキ表示と `FIRST PLAY` パネルを優先する。初回チュートリアル完了後も初回完走まではミッション類を出さず、3完走後はミッション画面に詳細を分離する。

### P1 リリース/保護者目線: SDK未接続の広告・購入導線が本番購入に見えるリスク
- 見える症状: パック画面の石付与やスタミナ回復が、実広告や実課金に見えるとストア審査・保護者説明で誤解を生む。
- 再現手順: パック画面とトップメニューのスタミナ回復ボタンを見る。
- なぜ重要か: 閉鎖テスト中でも「お金が発生するのか」「広告を見た扱いなのか」が曖昧だと信頼を損なう。
- 対応状況: 対応済み。パック画面に`CLOSED TEST`、`TEST AD`、`TEST GRANT`を表示し、トップメニューのスタミナ回復も`TEST`表示にした。本番リリース時はSDK成功コールバックへ置き換える。

### P1 プレイヤー目線: スマホ割り込みで60秒ランが失われるリスク
- 見える症状: 通知、ホーム移動、別アプリ確認、ゲーム画面外への移動で、残り時間が意図せず減ると理不尽に感じる。
- 再現手順: プレイ開始後にPAUSE、トップメニュー移動、ブラウザ/アプリ非表示を行う。
- なぜ重要か: スマホゲームでは短時間プレイほど割り込み耐性が信頼に直結する。スタミナ消費後の事故は離脱理由になりやすい。
- 対応状況: 対応済み。手動PAUSEと、画面外/非表示/pagehide/blur時の自動PAUSEを追加。再開操作までは時間、入力、スキル、リフレッシュ、演出更新を止める。ポーズ中に `やめる`、リザルトに `トップへ` を追加し、短時間プレイの離脱導線を明確にした。

### P1 教育者目線: 言葉が消えた後の学習文脈が弱い

- 見える症状: 「ぐんまけん」「だるま」などを完成しても、それが何なのかをゲーム内で振り返る場所が少ない。
- 再現手順: 1プレイを完了し、リザルト画面を見る。
- なぜ重要か: 親子・学校・地域学習文脈で紹介しにくく、単なる文字パズルに見える。
- 対応状況: 対応済み。全カードに `learnNote` を追加し、デッキ構築/コレクション、リザルト、共有画像に「ことばメモ」を表示する。デッキ構築側の絞り込みは、山/温泉などのテーマ分類ではなく `ALL/所持/未所持/GGG/GG/G`、`文字ALL/3字/4字/5字+`、`おすすめ/レア/名前` の実用軸へ変更した。ゲーム内のことばメモは、かな分解、地域メモ、文字数/スキル/所持状態、直接アクションに絞り、開発側の `話題/学び/配信/視聴` ラベルは表示しない。
- 次の改善候補: 地域メモを深掘りする場合も、ゲーム内では「探す」「編成する」「覚える」に役立つ短い文へ圧縮し、企画側の狙い説明にならないようにする。

### P1 配信者目線: 結果画面とプレイ中に話題のフックが必要

- 見える症状: スコアとベストワードだけだと、配信中に話す材料が短く終わる。プレイ中も視聴者へ今日の目標を伝えにくい。
- 再現手順: プレイ開始からリザルトまでを配信画面として見る。
- なぜ重要か: 配信者が自然にコメントを拾ったり、地域話題に広げたりしにくい。
- 対応状況: 対応済み。リザルトに短い「ことばメモ」を出し、共有文にも入れる。配信専用モードは通常プレイ導線を邪魔しやすいため削除し、ワードコール、通常リザルト、共有画像に見どころを集約した。
- 次の改善候補: 結果画面の共有画像や、ワードコールの読みやすさをさらに磨く。

### P2 プレイヤー目線: 日課が増えても圧に見えない整理が必要

- 見える症状: トップメニューにスタミナ、語数デイリー、スコア目標、連続ボーナス、石が並ぶ。
- 再現手順: 初回起動後にトップメニューを確認する。
- なぜ重要か: 初心者が「毎日やらなきゃ」と感じると、気軽な1プレイ導線が弱くなる。
- 対応状況: 対応済み。初回チュートリアル前は日課/スコア目標/今日のことば/今日の差し入れを隠し、チュートリアル後も初回完走まではミッション類を出さない。初回完走後に今日のことば/差し入れ、3完走後に日課/スコア目標/週替わりを出すが、詳細はミッション画面へ分離する。390pxスクショでは重なりなし。
- 次の改善候補: 日課達成済みホームが、翌日再訪時に「また1回だけ遊ぶ」導線として気持ちよく見えるかを継続確認する。

### P2 プレイヤー目線: スタミナ残数が文字だけだと見落としやすい

- 見える症状: STARTできない理由が、スタミナ文言だけだと一瞬で伝わりにくい。
- 再現手順: スタミナ5、4、0の状態でトップメニューとゲーム開始ボタンを見る。
- なぜ重要か: スタミナ制は納得感が大事。残りプレイ回数が明確なら、広告回復や待ち時間の意味も説明しやすい。
- 対応状況: 対応済み。トップメニューに5個の焼きまんじゅう風ピップを追加し、プレイ開始で1つ減る。0の時は焼き上がりまでの時間、報酬なし練習導線、空ピップを表示し、回復待ちでも操作学習を続けられるようにした。

### P1 プレイヤー/配信者目線: スタミナ切れで完全停止すると熱が途切れる
- 見える症状: 0スタミナ時に遊べなくなると、練習したい人や配信中の再挑戦が止まる。
- 再現手順: スタミナを0にしてゲーム画面の開始ボタンとリザルトの再挑戦ボタンを見る。
- なぜ重要か: スタミナは継続率と収益導線を作れる一方、完全停止は離脱や配信テンポ低下につながる。報酬なし練習ならゲーム理解は進み、経済バランスは壊れにくい。
- 対応状況: 対応済み。焼きまんじゅう0時のホームに`焼きまんじゅう切れ`案内と`練習する`ボタンを出し、開始ボタンも`練習`にする。練習リザルトは`PRACTICE`と`報酬なし`を表示、石/ベスト/デイリー/完走数を増やさない。

### P2 視聴者/アクセシビリティ目線: 色だけの識別は配信縮小や色覚差で弱くなる

- 見える症状: デッキ色が分かれていても、縮小視聴や端末差でカード由来のパネルを見間違える可能性がある。
- 再現手順: 設定でパネル番号をONにして、390px幅と縮小視聴相当の盤面を見る。
- なぜ重要か: プレイヤーが任意で補助表示を選べると、見やすさと画面の華やかさを両立できる。
- 対応状況: 対応済み。設定にパネル番号ON/OFFを追加し、ON時はデッキカードと盤面パネルへ1/2/3の小さな番号を表示する。

### P2 視聴者目線: パネルの色と文字は読めるが、今何が起きたかの説明が短い

- 見える症状: 消去演出はあるが、視聴者が一瞬で「何の言葉が完成したか」を追いきれない場合がある。
- 再現手順: 自動消去が複数回続く状態を横から見る。
- なぜ重要か: 見ているだけの人は入力の手触りを共有できないため、言葉完成の読み上げ情報が重要。
- 対応状況: 対応済み。消去時に完成語、得点、コンボを短時間だけ表示する「ワードコール」演出を追加。トップバー直下の盤面外表示帯に収め、パネル視界と次の操作を守る。リザルトにも見どころワードと次の目標をまとめ、未プレイ視聴者にも締めの意味が伝わりやすくした。
- 次の改善候補: リザルト共有画像とワードコールの文字サイズを実機で詰める。

### P2 4視点共通: 毎日の狙いがないと復帰理由と言葉の話題が弱い

- 見える症状: 日課や石報酬はあるが、「今日はどの言葉を見つけるか」という軽い目的がトップメニューで分かりにくい。
- 再現手順: 初回完走後にトップメニューを見る。
- なぜ重要か: プレイヤーは次の1プレイの目的、教育者は一言メモ、配信者は話題の入口、視聴者は応援対象をすぐ把握できるほうが継続理由になりやすい。
- 対応状況: 対応済み。初回完走後のミッション画面に `今日のことば` を追加し、日付と現在デッキから1枚を選んで表示する。報酬や出現率には影響させず、該当カードのことば図鑑へ移動できる導線にした。開始前の邪魔にならないよう、完走前は隠す。
- 次の改善候補: 閉鎖テストで、今日のことばが「遊びたくなる」「親子で話しやすい」「配信でコメントしやすい」に効いたかをクイック感想や自由記述で確認する。

- `scripts/game.js` / `index.html` / `styles.css`: 開始前のミッション圧を下げるため、初回完走前は今日のことば/差し入れ/日課/週目標を隠し、焼きまんじゅう型スタミナ、途中終了、リザルトからトップへ戻る導線を追加。

## リリース前に再確認すること

- メモ文は一般的な地域説明に留め、公式キャラ、ロゴ、写真、地図、第三者IPに依存しない。
- 教育要素は「学習補助」であり、正確性が必要な百科事典的記述に寄せすぎない。
- 配信画面では、リザルトのメモが小さすぎないか、OBSで縮小表示した状態でも読めるか確認する。
- 視聴者向けには、ワードコールが連鎖時に読み切れる長さか、配信画面の縮小表示で確認する。
- 配信専用モードは削除済みのため、ワードコールと通常リザルトだけで見どころが伝わるか確認する。

## 2026-06-12 追加デバッグ: 効果音テスト導線

- プレイヤー目線: 設定画面で効果音を事前に確認できるため、端末音量やサウンドOFFに気づかないまま遊び始める不安を減らせる。

- 視聴者目線: 完成時の効果音が聞こえると、盤面を凝視していなくても成功タイミングが伝わりやすい。

## 2026-06-12 追加デバッグ: 週替わり祭りチャレンジ

- プレイヤー目線: デイリーを終えた後も「今週あと何語」という軽い継続理由ができる。初回とウォームアップ中は隠すため、始めた直後の圧は増やさない。
- 教育者目線: 1日単位ではなく週単位で地域のことばを集める導線になり、親子や授業で少しずつ触る理由を作れる。
- 配信者目線: 配信中に「今週の祭りを進める」と言いやすく、単発のスコア挑戦以外の話題を作れる。
- 視聴者目線: 配信を途中から見ても、週目標の進み具合で現在の目的が分かりやすい。報酬は石+1に抑え、有料有利には見せない。

## 2026-06-12 追加デバッグ: リザルト進捗リキャップ

- プレイヤー目線: プレイ後に日課と週目標の進み具合がすぐ分かるため、スタミナ消費後でも「あと少しだけ」の理由を作れる。
- 教育者目線: 1プレイの結果を、今日の学習量と今週の継続目標に結びつけられる。練習モードでは報酬なしを明示し、記録との混同を避ける。
- 配信者目線: リザルトで次の挑戦、今日の達成、週の進捗を短く読めるため、締めコメントや次枠への導線を作りやすい。
- 視聴者目線: スコア以外に「今どこまで進んだか」が見えるため、途中から見ても状況を追いやすく、応援コメントのきっかけになる。

## 2026-06-12 追加デバッグ: 非PIIイベントログつきテストレポート

- プレイヤー目線: 「何を押したか」を思い出せなくても、直近の画面遷移、プレイ開始/終了、パック、設定変更がレポートに入るため、不具合報告の負担が下がる。
- 教育者目線: 授業や親子利用で、どの設定や練習モードを使ったかを匿名の操作事実として確認でき、学習導線の改善に使える。
- 配信者目線: 配信中のトラブル時に、共有、音確認、リザルト周辺の操作が時系列で残り、再現確認がしやすい。
- 視聴者目線: 視聴者が見た違和感と実際の操作ログを突き合わせやすくなり、未プレイ視聴者からの報告も検証しやすい。
- リリース目線: 個人情報を増やさず、最大24件に丸めた閉鎖テスト用ログとして扱う。本番分析やクラッシュ収集へ進む前の設計メモになる。

## 2026-06-12 追加デバッグ: リザルトクイック感想

- プレイヤー目線: 1プレイ直後に「楽しい」「難しい」をすぐ残せるため、面白さと詰まりの温度を忘れる前に拾える。
- 教育者目線: 「学びあり」を独立させ、ことばメモや地域題材が単なる装飾ではなく学習フックになっているか確認できる。
- 配信者目線: 「盛り上がる」を用意し、ワードコール、リザルト、開封演出が画面映えしているか閉鎖テスターから拾いやすくした。
- 視聴者目線: 「見づらい」を短い選択肢として置くことで、文字サイズ、色、演出の読みづらさを低負担で報告できる。
- リリース目線: 感想は最大12件の端末内保存と `quick_feedback` の非PIIイベントに限定し、テストレポートへ直近要約を同梱する。

## 2026-06-12 追加デバッグ: ホームのLAST RUN導線

- プレイヤー目線: 再起動後も前回スコア、見どころワード、次の目標がトップメニューに残るため、「昨日の続きで1回だけ」が始めやすい。
