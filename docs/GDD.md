# ぐんもじぱずる 試作直行GDD

## 目的

`ぐんもじぱずる` は、60秒で遊べるスマホ縦持ち専用の文字パズルです。四角いかなパネルを上下左右にスライドして動かし、隣接したパネルの文字セットがデッキ内カードの名称と一致すると自動で消えてスコアを獲得します。

MVPでは配信機能を入れず、1人プレイの手触り、カードデッキ、60秒スコア、パック入手の基礎だけを検証します。
リザルトでは今回のスコア、ランク、見どころワード、獲得したGだるまと達成理由を短く表示し、練習時は報酬なしを明示します。リザルト後の導線は `もう1回` / `トップへ` / `デッキ構築` の3つに絞ります。
設定ではサウンド、振動、演出控えめ、大きめ文字、高コントラスト、パネル番号表示を切り替えられます。
リザルトではスコア、ランク、見どころワード、報酬サマリーを通常表示の中でまとめ、次の狙いや振り返りはプレイヤー自身に委ねます。

## コアルール

- 旧配信切替ボタン、配信モード切替、専用情報帯、配信専用リザルトカードは削除し、通常プレイの導線を優先します。
- 初回ユーザーは `tutorialComplete=false` の間、トップメニューで日課/スコア目標/今日のことば/今日の差し入れを非表示にし、デッキ表示と `FIRST PLAY` ボタンを優先します。初回チュートリアル完了後も、初回完走まではミッション類を出さず、まず通常プレイを始める導線に集中させます。初回完走後に `今日のことば` をミッション画面で解放し、3完走後に日課/スコア目標/週替わりもミッション画面で通常表示します。今日の差し入れは完走直後ではなく次回ログイン時だけモーダルで出し、リザルト後の再挑戦、デッキ構築、パック確認の動線を塞ぎません。通常ホームにはミッション要約ボタンだけを置き、詳細カードはミッション画面に分離します。
- 初回完走後のミッション画面では、日付と現在デッキから決まる `今日のことば` を1枚表示します。報酬や出現率には影響せず、プレイヤーの次の狙い、教育者の一言メモ、配信者/視聴者の会話材料として扱います。ただしゲーム内には `話題` や `配信` など開発側の狙いラベルを出さず、かな分解、文字数、スコア目安、地域メモだけを見せます。`図鑑`ボタンから該当カードのことばメモへ移動できます。
- 初回後のトップメニューでは、保存された直近リザルトから前回スコアと見どころワードを `LAST RUN` として短く表示し、再訪時に次の1プレイへ戻りやすくします。
- 初回プレイ中は `FIRST CLEAR` 案内で光る2枚のスライドを促し、迷った場合は8秒後にお手本スライドで `ぐんまけん` を完成させます。演出控えめ設定でも静的なヒントは残します。

- デイリースコア目標は1日1回、5000点到達でGだるま+1を自動付与します。日付が変わると進捗と受け取り状態をリセットします。
- 1プレイは60秒。
- プレイ中はPAUSEボタンで一時停止できます。ポーズ中は再開または途中終了を選べます。アプリが非表示になった時、ページ離脱時、ゲーム画面外へ戻った時も自動で一時停止し、プレイヤーの再開操作まで時間、入力、スキル、リフレッシュ、演出更新を止めます。
- 1プレイ開始時に焼きまんじゅう型スタミナを1本消費します。初期値は最大5本、10分で1本焼き上がります。ゲーム内表示は「焼きまんじゅう」に寄せ、内部仕様としてはスタミナ制として扱います。
- 焼きまんじゅう残量はトップメニュー上部のステータス帯に表示します。プレイ可否に関わる情報なので、デッキ、メインボタン、広告枠より先に見える位置を基本とします。
- スタミナ不足時は報酬なしの練習プレイを開始できます。練習プレイのスコア、語数、ベスト、デイリー目標、Gだるま報酬、完走数は保存対象にしません。
- 焼きまんじゅう0のトップメニューでは、焼き上がりまでの時間と報酬なし練習導線を短く表示し、プレイヤーが待つ/練習する/テスト回復する状態を混同しないようにします。
- 広告視聴想定の回復導線として、トップメニューからスタミナを1回復できます。MVPでは即時成功の仮実装で、実リリース時は広告SDKの成功コールバックに置き換えます。
- デイリー目標は1日1回、3プレイ相当で達成できる日替わりお題として、プレイ回数/語数/合計スコアのいずれかを出します。達成するとGだるま+1を自動付与し、日付が変わると進行状況とお題をリセットします。
- 今日の差し入れは1日1回、初回完走後の次回ログイン時にモーダルでGだるま+1を手動受け取りできます。初回完走前と完走直後は表示せず、受け取り後は通常ホームに常駐させません。
- デイリー目標を3日連続で達成すると、ストリークボーナスとしてGだるま+1を追加付与します。1日途切れた場合は連続日数を0に戻します。
- 週替わり祭りチャレンジは、月曜始まりの1週間ごとに20プレイ相当の回数/語数/合計スコアお題を出し、達成するとGだるま+10を自動付与します。初回とウォームアップ中は表示せず、ミッション画面で継続目標として見せます。練習プレイは進捗対象外です。
- デイリーランキングは毎日0時に切り替わる気軽な競争枠として、トップ10と自分のスコアをランキング画面に表示します。翌日は昨日ランキングを見られ、トップ10入りしていた場合はGだるま+3を自動配布します。
- リザルトでは、スコア、ランク、見どころワード、報酬サマリーを短く表示します。練習プレイでは報酬なしで進捗しないことが分かる文言にします。リザルト後は `もう1回` / `トップへ` / `デッキ構築` を迷わず選べます。
- デッキは3枚。将来の増減に備え、データ側の`deckSize`で管理します。
- デッキカードには色を持たせますが、プレイヤー向けのデッキ状態は `推し` と `デッキ入り` だけにします。盤面では `パネル番号表示` をONにすると、色だけに頼らず小番号でパネルを見分けられます。
- ベーシックデッキは全ユーザー配布: `ぐんまけん`, `だるま`, `あかぎさん`。
- デッキ内カード名の読みを全文字単位で結合し、盤面かなの出現プールを作ります。
- 重複文字は重複分だけ出現率が高くなります。ベーシックデッキでは13文字、`ん`は3/13、`ま`は2/13です。
- スライド、リフレッシュ、スキル発動後に盤面を自動判定します。隣接してつながったパネルの文字セットがデッキ内カード名と一致していれば消去成功です。文字の順番は問いません。
- 消去成功時は、配信者と視聴者が追いやすいように完成語、得点、コンボを短時間だけ表示するワードコールを出します。ワードコールはトップバー直下の盤面外表示帯に寄せ、通常消去のCanvas内加点文字は出さず、次のスライド操作とパネル視認性を優先します。
- 汎用トーストはプレイ中の盤面には出さず、メニュー、結果、設定など操作を止めても問題ない画面で表示します。
- 配信専用の情報帯は使わず、完成語と得点のワードコールだけを盤面外に短時間表示します。
- 効果音はWeb Audioで再生します。設定画面の`SOUND 音を確認`で、プレイ前や録画前に端末の音量、サウンド設定、ブラウザの音声開始状態を確認できます。
- 閉鎖テスト中は、起動、画面遷移、設定変更、プレイ開始/終了、パック開封、スタミナ回復などを最大24件の非PIIイベントとして端末内に保存します。ゲーム内にはテスター向けの感想入力やフィードバック導線を出しません。
- パネルは6列×7行のグリッド上で、上下左右へスライドして隣のパネルと入れ替えます。
- スコア式は`30 × 文字数^3`。短い言葉は完成頻度とコンボ、長い言葉は1語あたりの高得点で差別化します。
- リフレッシュボタンで盤面を全リセットできます。初期仕様は5秒クールダウン、スコア減点なし、スキルゲージ増加なしです。

## カードとスキル

- 推しカードを1枚設定します。プレイ中にゲージ100%で、デッキ3枚のスキルがまとめて発動します。推しは中央表示とゲージ上昇の個性として扱います。
- カードデータは`id`, `nameKey`, `displayName`, `readingKana`, `learnNote`, `guidePrompt`, `guideLearning`, `guideReactionCue`, `guideViewerCue`, `rarityG`, `seasonId`, `skillId`, `skillPower`, `artKey`, `sourceNote`, `rightsStatus`を持ちます。
- `learnNote`はデッキ構築/コレクション画面やミッション画面で表示し、カード選択や地域名の理解に役立てます。リザルトではプレイ後の導線を優先し、詳細メモへは直接誘導しません。
- コレクション画面では、ゲーム上の判断に直結する `ALL`, `所持`, `未所持`, `GGG`, `GG`, `G` で絞り込めます。山/温泉などの地域カテゴリは内部メモとして残せますが、デッキ構築の主UIには出しません。
- コレクション画面では、`文字ALL`, `3字`, `4字`, `5字+` で文字数フィルタを切り替え、`おすすめ`, `レア`, `名前` でソートできます。おすすめは編成中、所持、レア度、文字数を優先し、デッキに入れたいカードへ早く到達するための並びにします。
- ことばメモでは、選択カードの読みを1文字ずつ分解し、地域メモ、文字数/スキル/所持状態、直接アクションを表示します。カードデータ上の `guidePrompt`, `guideLearning`, `guideReactionCue`, `guideViewerCue` は企画・テストレポート用の内部メモとして保持できますが、ゲーム内UIには `話題`, `学び`, `会話`, `視聴` ラベルを表示しません。
- ことば図鑑の下部には状態に応じたアクションボタンを表示します。デッキ内カードは推し設定、所持カードはデッキ投入、未所持カードは選択チケット入手または引換だるま交換へつなげます。
- 同じ`nameKey`のカードはデッキに1枚だけ入れられます。
- 重複所持は自動で凸になり、1凸ごとにスキル効果だけが2%上昇します。上限は10凸です。
- 仮カードフレームはコード描画です。画像生成素材に置き換える場合も、プロンプトと権利メモを保存し、公式キャラ、ロゴ、写真、既存地図、第三者IPを混ぜません。

## 特殊アイテム

- 現行版では特殊アイテムパネルを出現させません。
- 盤面の読みやすさと操作テンポを優先し、主役をことば消去、リフレッシュ、デッキスキルに集約します。
- 将来復活させる場合は、存在感、操作理由、スコアへの影響が明確な形に再設計します。
- 初期効果は以下です。
  - `お邪魔`: 消えるだけで効果なし。
  - `3秒延ばす`: 残り時間を3秒増やします。
  - `スコアアップ`: 8秒間スコアを30%増やします。

## パック

- パックは期間限定シーズン制です。各パックに `saleStartsAt` / `saleEndsAt` を持たせ、販売期間外は開封できません。
- 第1弾パックは `ぐんまのし` です。現在は固定カードリストですが、データ上は将来の `randomizable-card-pool` 化を見越して `entryMode` を明示します。
- 広告視聴または課金でゲーム内アイテムを獲得し、アイテム10個で1パックを購入します。
- 現在の静的HTML試作では広告/課金SDK未接続のため、パック画面の石付与は`CLOSED TEST`表示つきのテスト導線として扱います。本番購入は発生しません。
- 価格目安は広告10回分、または日本円100〜200円程度です。
- 1パックの中身は1枚、または選択チケット1枚です。
- レア度は星の代わりにGアイコンで表示します。`GGG`合計4%で2種類、`GG`合計25%で4種類、`G`合計70%で6種類、選択チケット1%です。
- `ぐんまのし` のカードは `GGG`: まえばし/たかさき、`GG`: きりゅう/いせさき/おおた/ぬまた、`G`: たてばやし/しぶかわ/ふじおか/とみおか/あんなか/みどり です。
- パック画面には、開封前に第1弾12種のカードラインナップを表示します。各カードは個別排出率と所持状態を持ち、タップするとデッキ構築/ことばメモへ移動します。
- パック画面には、第1弾の所持数、進捗バー、次に狙う未所持カードを表示します。これは収集目標の見える化であり、スコア有利報酬や有料限定報酬にはしません。
- パック開封後のリワード表示には、当たったカードのことばメモ、または選択チケットで取れる未所持カード候補へ直接移動するボタンを表示します。
- パック開封ごとに引換だるまを1個付与します。引換だるま50個で、第1弾パック内の未所持カード1枚をコレクションから選んで入手できます。
- 所持済みカードはデッキ構築画面で選択中スロットに入れ替えられます。同名 `nameKey` は1デッキに1枚までです。
- 全カードは無料入手可能です。有料購入は時短扱いにし、有料限定でスコア有利になるカードは禁止します。
- コンプガチャ型の報酬、公式IPに見えるカード絵、商標リスクの高い店名・商品名はMVPに入れません。

## 試作検証

- 60秒で初心者が最低1回、慣れた人が複数回カード名を完成できるか。
- 四角パネルのスライド移動、順番自由の自動消去、リフレッシュ、デッキスキルが安定するか。
- デッキ出現プールが重複文字込みの確率で生成されるか。
- パック排出率、選択チケット、重複凸、同名カードのデッキ制限が仕様通りか。
- パック画面のコレクション進捗が、所持数、進捗バー、次の未所持カードを正しく更新するか。
- パック開封後のアクションボタンが、カード入手時は該当カード、選択チケット時は未所持候補のことばメモへ遷移するか。
- パック開封で引換だるまが1個増え、50個時に未所持カードだけを交換でき、交換後に0へ戻るか。
- コレクションの `ALL/所持/未所持/GGG/GG/G` 絞り込み、`文字ALL/3字/4字/5字+` 文字数フィルタ、`おすすめ/レア/名前` ソートが、選択カード、所持状態、デッキ投入可否と矛盾なく更新されるか。
- ことばメモのかな分解、地域メモ、文字数/スキル/所持状態、直接アクションが選択カードに合わせて更新され、`話題/学び/会話/視聴` など開発側の狙いラベルがゲーム内に出ていないか。
- ことば図鑑のアクションボタンが、推し設定、所持カード投入、選択チケット、引換だるま、未所持disabled状態を正しく出し分けるか。
- 今日のことばが初回完走前は隠れ、初回完走後は現在デッキ内のカードから日替わりで表示され、図鑑ボタンで該当カードのメモへ移動できるか。
- 焼きまんじゅうの消費、自然回復、広告視聴想定の+1本回復、保存後の復元、不足時の報酬なし練習プレイが仕様通りか。
- 焼きまんじゅう0のトップメニューで、焼き上がりカウント、報酬なし練習文言、練習ボタンが横スクロールなしで読めるか。
- リザルトの報酬サマリーが、今回獲得したGだるま、達成理由、練習時の報酬なしを正しく表示するか。
- リザルトの主ボタンが `もう1回` / `トップへ` / `デッキ構築` の3つだけで、上側が見切れずトップメニューと近い雰囲気になっているか。
- セーブ復元後のトップメニューで `LAST RUN` が表示され、前回スコアと見どころワードが横スクロールなしで読めるか。
- デイリー目標の進行、日付リセット、達成報酬のGだるま+1、3日連続ボーナスのGだるま+1が仕様通りか。
- 今日の差し入れが初回完走前と完走直後は隠れ、次回ログインモーダルで1回だけGだるま+1を受け取れ、受取後にホームの縦幅を圧迫しないか。
- 週替わり祭りチャレンジの進行、週リセット、達成報酬のGだるま+10、練習プレイ除外が仕様通りか。
- リザルトに次の目標、進捗チップ、フィードバック、共有、メモ導線が出ていないか。プレイ中の `やめる` とリザルトの `トップへ` が迷わず使えるか。
- 設定画面の音確認ボタンで効果音が鳴り、サウンドOFF時はOFF状態が分かるか。
- 非PIIの最近イベントが端末内に保持され、将来の外部テスト導線へ拡張できるか。
- カードデータに読み仮名、出典メモ、権利ステータスが揃っているか。
